§ 00 — Motivación

El problema de los modelos generativos

Dado un dataset \(\mathcal{D} = \{\mathbf{x}^{(i)}\}_{i=1}^N\), queremos aprender una distribución \(p_\theta(\mathbf{x})\) tal que podamos:

  • Evaluar la densidad de nuevos datos (detección de anomalías, compresión)
  • Generar nuevas muestras realistas
  • Aprender representaciones en un espacio latente estructurado

El autoencoder clásico aprende una representación comprimida pero no es generativo: el espacio latente no tiene estructura probabilística, y muestrear de él produce basura. Los VAEs resuelven esto exactamente.

Un VAE no aprende un punto en el espacio latente por imagen — aprende una distribución gaussiana en el espacio latente por imagen. Esto fuerza que el espacio latente sea continuo y estructurado, haciendo posible la generación.

Autoencoder Puntos arbitrarios No nativa Sin estructura probabilística
VAE Distribuciones gaussianas Sí, muestreando \(\mathbf{z} \sim p(\mathbf{z})\) Imágenes algo borrosas

§ 01 — Modelos con Variables Latentes

El modelo probabilístico

Un VAE es un modelo generativo con variables latentes. Postulamos que cada dato observado \(\mathbf{x}\) fue generado por un proceso de dos pasos:

Proceso generativo $$\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(0, \mathbf{I}) \qquad \text{(prior sobre latentes)}$$ $$\mathbf{x} \sim p_\theta(\mathbf{x} \mid \mathbf{z}) \qquad \text{(likelihood parametrizada por red neuronal)}$$

La distribución conjunta del modelo es:

$$p_\theta(\mathbf{x}, \mathbf{z}) = p_\theta(\mathbf{x} \mid \mathbf{z})\, p(\mathbf{z})$$

El encoder \(q_\phi(\mathbf{z} \mid \mathbf{x})\) (también llamado red de inferencia o approximate posterior) mapea cada dato a una distribución gaussiana en el espacio latente.

El decoder \(p_\theta(\mathbf{x} \mid \mathbf{z})\) (red generativa) mapea cada punto latente de regreso al espacio de los datos.

Distribuciones paramétricas $$q_\phi(\mathbf{z} \mid \mathbf{x}) = \mathcal{N}\!\left(\mathbf{z};\; \boldsymbol{\mu}_\phi(\mathbf{x}),\; \text{diag}(\boldsymbol{\sigma}^2_\phi(\mathbf{x}))\right)$$ $$p_\theta(\mathbf{x} \mid \mathbf{z}) = \begin{cases} \mathcal{N}(\boldsymbol{\mu}_\theta(\mathbf{z}), \sigma^2 \mathbf{I}) & \text{datos continuos} \\ \text{Bernoulli}(\boldsymbol{\pi}_\theta(\mathbf{z})) & \text{datos binarios / imágenes normalizadas} \end{cases}$$

Ambas redes se entrenan conjuntamente — los parámetros \(\phi\) del encoder y \(\theta\) del decoder son optimizados al mismo tiempo con el mismo objetivo.

§ 02 — El Problema de la Intratabilidad

¿Por qué no podemos maximizar la likelihood directamente?

El objetivo natural es maximizar la log-verosimilitud marginal de los datos:

$$\log p_\theta(\mathbf{x}) = \log \int p_\theta(\mathbf{x} \mid \mathbf{z})\, p(\mathbf{z})\, d\mathbf{z}$$

Esta integral es intratable por dos razones fundamentales:

El espacio latente tiene dimensión \(d_z\) (típicamente 64–512). La integral sobre un espacio continuo de alta dimensión no tiene solución analítica para redes neuronales no lineales. La cuadratura numérica es exponencial en \(d_z\).
Tampoco podemos calcular el posterior verdadero \(p_\theta(\mathbf{z} \mid \mathbf{x})\) — que necesitaríamos para entrenar el encoder — ya que por Bayes: $$p_\theta(\mathbf{z} \mid \mathbf{x}) = \frac{p_\theta(\mathbf{x} \mid \mathbf{z})\,p(\mathbf{z})}{p_\theta(\mathbf{x})}$$ y el denominador es exactamente la integral intratable de arriba.

Este es el problema central que VAE resuelve: inferencia aproximada del posterior mediante una red neuronal, entrenada junto con el modelo generativo.

§ 03 — Inferencia Variacional

Aproximar el posterior con optimización

La idea de la inferencia variacional (VI) es reemplazar el problema de inferencia por un problema de optimización. En lugar de calcular \(p_\theta(\mathbf{z} \mid \mathbf{x})\) exactamente, introducimos una familia de distribuciones \(\mathcal{Q}\) y buscamos el miembro más cercano al posterior verdadero:

Objetivo de la inferencia variacional $$q^*_\phi(\mathbf{z} \mid \mathbf{x}) = \arg\min_{q_\phi \in \mathcal{Q}}\; D_\text{KL}\!\left(q_\phi(\mathbf{z} \mid \mathbf{x})\;\|\;p_\theta(\mathbf{z} \mid \mathbf{x})\right)$$

En VI clásica, \(\mathcal{Q}\) es una familia paramétrica simple y los parámetros se optimizan por separado para cada dato \(\mathbf{x}^{(i)}\). Esto es costoso (\(O(N)\) problemas de optimización).

La innovación de los VAEs (Kingma & Welling, 2014) es usar una red neuronal (el encoder \(q_\phi(\mathbf{z}|\mathbf{x})\)) para amortizar la inferencia: los mismos parámetros \(\phi\) sirven para todos los datos. Se llama Amortized Variational Inference.

En VI clásica: optimizar \(q_i\) separadamente para cada \(\mathbf{x}^{(i)}\) — \(O(N)\) optimizaciones.

En VAE: el encoder \(q_\phi\) aprende a mapear directamente cualquier \(\mathbf{x}\) a los parámetros de su posterior aproximado — una sola optimización con gradiente estocástico.

¿Por qué minimizar la KL-divergencia es difícil directamente?

El obstáculo $$D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,p_\theta(\mathbf{z}|\mathbf{x})) = \mathbb{E}_{q_\phi}\!\left[\log\frac{q_\phi(\mathbf{z}|\mathbf{x})}{p_\theta(\mathbf{z}|\mathbf{x})}\right]$$ $$= \mathbb{E}_{q_\phi}\!\left[\log q_\phi(\mathbf{z}|\mathbf{x}) - \log p_\theta(\mathbf{x}|\mathbf{z}) - \log p(\mathbf{z})\right] + \log p_\theta(\mathbf{x})$$

El término \(\log p_\theta(\mathbf{x})\) es intratable — ¡estamos de vuelta al inicio! La solución es derivar el ELBO, que evita calcular \(p_\theta(\mathbf{x})\).

§ 04 — El ELBO

Evidence Lower Bound

Derivamos el ELBO partiendo directamente de \(\log p_\theta(\mathbf{x})\). La clave es multiplicar y dividir por \(q_\phi(\mathbf{z}|\mathbf{x})\) dentro de la integral:

Derivación vía Jensen's Inequality $$\log p_\theta(\mathbf{x}) = \log \int p_\theta(\mathbf{x}, \mathbf{z})\, d\mathbf{z}$$ $$= \log \int q_\phi(\mathbf{z}|\mathbf{x})\, \frac{p_\theta(\mathbf{x}, \mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\, d\mathbf{z}$$ $$= \log\; \mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}\!\left[\frac{p_\theta(\mathbf{x}, \mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\right]$$ $$\geq\; \mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}\!\left[\log\frac{p_\theta(\mathbf{x}, \mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\right] \quad \leftarrow \textit{Jensen: } \log\mathbb{E}[X] \geq \mathbb{E}[\log X]$$ $$=:\; \mathcal{L}(\theta, \phi;\, \mathbf{x}) \qquad \text{(ELBO)}$$

Relación exacta entre ELBO y log-likelihood

La brecha entre \(\log p_\theta(\mathbf{x})\) y el ELBO no es arbitraria — tiene un significado preciso:

Identidad fundamental $$\log p_\theta(\mathbf{x}) = \underbrace{\mathcal{L}(\theta, \phi;\, \mathbf{x})}_{\text{ELBO}} + \underbrace{D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,p_\theta(\mathbf{z}|\mathbf{x}))}_{\geq\, 0}$$
Demostración de la identidad $$\mathcal{L} = \mathbb{E}_{q_\phi}\!\left[\log\frac{p_\theta(\mathbf{x}, \mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\right] = \mathbb{E}_{q_\phi}\!\left[\log\frac{p_\theta(\mathbf{z}|\mathbf{x})\,p_\theta(\mathbf{x})}{q_\phi(\mathbf{z}|\mathbf{x})}\right]$$ $$= \log p_\theta(\mathbf{x}) + \mathbb{E}_{q_\phi}\!\left[\log\frac{p_\theta(\mathbf{z}|\mathbf{x})}{q_\phi(\mathbf{z}|\mathbf{x})}\right] = \log p_\theta(\mathbf{x}) - D_\text{KL}(q_\phi \| p_\theta(\cdot|\mathbf{x}))$$ $$\implies \log p_\theta(\mathbf{x}) = \mathcal{L} + D_\text{KL}(q_\phi\|p_\theta(\cdot|\mathbf{x}))\;\;\blacksquare$$

Como \(D_\text{KL} \geq 0\) siempre, el ELBO es una cota inferior de la log-likelihood. Maximizar el ELBO sirve dos propósitos simultáneamente:

  • Empuja hacia arriba \(\log p_\theta(\mathbf{x})\) — mejora el modelo generativo
  • Reduce \(D_\text{KL}(q_\phi \| p_\theta(\cdot|\mathbf{x}))\) — mejora el encoder
§ 05 — Los Dos Términos del ELBO

Reconstrucción y regularización

Expandiendo el ELBO usando la descomposición \(p_\theta(\mathbf{x}, \mathbf{z}) = p_\theta(\mathbf{x}|\mathbf{z})\,p(\mathbf{z})\):

Descomposición del ELBO $$\mathcal{L}(\theta, \phi;\, \mathbf{x}) = \underbrace{\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}\!\left[\log p_\theta(\mathbf{x} \mid \mathbf{z})\right]}_{\text{término de reconstrucción}} - \underbrace{D_\text{KL}\!\left(q_\phi(\mathbf{z}|\mathbf{x})\;\|\;p(\mathbf{z})\right)}_{\text{término de regularización}}$$
Derivación de la descomposición $$\mathcal{L} = \mathbb{E}_{q_\phi}\!\left[\log\frac{p_\theta(\mathbf{x}|\mathbf{z})\,p(\mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\right]$$ $$= \mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|\mathbf{z})] + \mathbb{E}_{q_\phi}\!\left[\log\frac{p(\mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})}\right]$$ $$= \mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|\mathbf{z})] - D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,p(\mathbf{z}))\;\;\blacksquare$$
\(\mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|\mathbf{z})]\) — Reconstruction Term

¿Qué mide? La log-verosimilitud esperada del dato \(\mathbf{x}\) bajo el decoder, promediada sobre el posterior aproximado del encoder. En otras palabras: ¿qué tan bien el decoder puede reconstruir \(\mathbf{x}\) a partir de las representaciones latentes que produce el encoder?

Forma explícita para datos continuos: $$\mathbb{E}_{q_\phi}\!\left[\log p_\theta(\mathbf{x}|\mathbf{z})\right] = -\frac{1}{2}\mathbb{E}_{q_\phi}\!\left[\|\mathbf{x} - \boldsymbol{\mu}_\theta(\mathbf{z})\|^2\right] + \text{const}$$ Es esencialmente la MSE entre input y reconstrucción.

Para datos binarios (Bernoulli likelihood): $$= \sum_j \left[x_j \log \pi_j + (1-x_j)\log(1-\pi_j)\right]$$ Es la binary cross-entropy entre input y reconstrucción.

Intuición: Este término empuja al encoder y decoder a funcionar bien como autoencoder. Sin el término KL, el encoder colapsaría en una función determinística (autoencoder estándar).

\(-D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,p(\mathbf{z}))\) — Regularization Term (KL Term)

¿Qué mide? La divergencia entre el posterior aproximado del encoder \(q_\phi(\mathbf{z}|\mathbf{x})\) y el prior \(p(\mathbf{z}) = \mathcal{N}(0, \mathbf{I})\). Este término se resta, así que maximizar el ELBO implica minimizar esta divergencia.

Forma cerrada (gaussianas): Como ambas distribuciones son gaussianas diagonales, la KL tiene solución analítica exacta — no necesita estimación Monte Carlo: $$D_\text{KL} = -\frac{1}{2}\sum_{j=1}^{d_z}\!\left(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2\right)$$

Intuición: Regula el espacio latente. Sin este término, el encoder aprendería a colapsar cada dato en un punto (varianza \(\to 0\)) — reconstrucción perfecta, espacio latente inútil para generación. El término KL fuerza que el posterior se parezca al prior: distribuciones compactas, centradas en el origen.

Posterior collapse: Si este término domina demasiado (β-VAE), el encoder ignora \(\mathbf{x}\) y devuelve siempre el prior — el latente pierde información.

La tensión fundamental del VAE

Reconstrucción: "Aprende un código que permita reconstruir el input perfectamente."

KL (regularización): "Pero ese código debe parecerse al prior gaussiano."

El entrenamiento balancea estas dos fuerzas opuestas. El resultado es un espacio latente suave y estructurado: puntos cercanos en el espacio latente corresponden a datos similares, y podemos interpolar o muestrear nuevas imágenes.

El parámetro β: controlando el balance

En el β-VAE (Higgins et al., 2017), se introduce un hiperparámetro \(\beta > 0\) que pondera el término KL:

$$\mathcal{L}_\beta = \mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|\mathbf{z})] - \beta\cdot D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,p(\mathbf{z}))$$

\(\beta = 1\) es el VAE estándar. \(\beta > 1\) fuerza mayor presión hacia representaciones disentangled (factorizadas). \(\beta < 1\) prioriza la reconstrucción.

§ 06 — El Truco de Reparametrización

Cómo hacer backprop a través de una muestra

El término de reconstrucción requiere computar:

$$\nabla_\phi\; \mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}\!\left[\log p_\theta(\mathbf{x}|\mathbf{z})\right]$$

El problema: la distribución de la que muestreamos, \(q_\phi\), depende de \(\phi\) — los parámetros que queremos optimizar. No podemos simplemente mover el gradiente dentro de la esperanza.

El estimador naive (REINFORCE): $$\nabla_\phi\,\mathbb{E}_{q_\phi}[f(\mathbf{z})] \approx \frac{1}{S}\sum_{s} f(\mathbf{z}^{(s)})\,\nabla_\phi\log q_\phi(\mathbf{z}^{(s)})$$ Esta estimación es correcta pero tiene varianza muy alta — el entrenamiento no converge.

La solución: reparametrizar el muestreo

La idea central es separar el ruido de los parámetros. En lugar de muestrear \(\mathbf{z}\) directamente de \(q_\phi(\mathbf{z}|\mathbf{x})\), expresamos \(\mathbf{z}\) como una función determinística de \(\phi\) y un ruido externo \(\boldsymbol{\epsilon}\):

Reparametrización para distribución gaussiana diagonal $$\mathbf{z} = \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}, \qquad \boldsymbol{\epsilon} \sim \mathcal{N}(0, \mathbf{I})$$

Ahora \(\mathbf{z}\) es una función diferenciable de \(\phi\), y podemos usar backpropagation normalmente:

Gradiente con reparametrización — bajo varianza $$\nabla_\phi\;\mathbb{E}_{q_\phi}[f(\mathbf{z})] = \nabla_\phi\;\mathbb{E}_{\boldsymbol{\epsilon}\sim\mathcal{N}(0,\mathbf{I})}\!\left[f(\boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon})\right]$$ $$= \mathbb{E}_{\boldsymbol{\epsilon}}\!\left[\nabla_\phi\, f(\boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon})\right]$$ $$\approx \frac{1}{S}\sum_{s=1}^S \nabla_\phi\, f\!\left(\boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon}^{(s)}\right), \quad \boldsymbol{\epsilon}^{(s)} \sim \mathcal{N}(0, \mathbf{I})$$

En la práctica se usa \(S=1\) — una sola muestra por forward pass — y la varianza es suficientemente baja gracias al promediado sobre el batch.

Con reparametrización, el grafo computacional fluye así:

x → Encoder → (μ, log σ²) ε ~ N(0,I) [sin gradiente]
↓ ↓
z = μ + σ·ε ← operación diferenciable
↓
Decoder → x̂ → Loss
El gradiente fluye libremente a través de la operación z = μ + σ·ε hacia los parámetros del encoder \(\phi\).

¿Por qué codificar log σ² y no σ directamente?

En la práctica, el encoder predice \(\log \sigma^2_j\) en lugar de \(\sigma_j\). Esto tiene dos ventajas: (1) \(\log \sigma^2\) puede tomar cualquier valor real mientras que \(\sigma > 0\) — sin restricciones de activación. (2) La forma cerrada de la KL se expresa naturalmente en términos de \(\log \sigma^2\).

§ 07 — Forma Cerrada de la KL

La KL gaussiana tiene solución analítica

Cuando \(q_\phi(\mathbf{z}|\mathbf{x}) = \mathcal{N}(\boldsymbol{\mu}, \text{diag}(\boldsymbol{\sigma}^2))\) y \(p(\mathbf{z}) = \mathcal{N}(0, \mathbf{I})\), la KL-divergencia se puede calcular exactamente sin muestreo:

KL cerrada — resultado esencial $$D_\text{KL}(q_\phi(\mathbf{z}|\mathbf{x})\,\|\,\mathcal{N}(0,\mathbf{I})) = -\frac{1}{2}\sum_{j=1}^{d_z}\!\left(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2\right)$$
Derivación para dimensión 1 (extensible a diagonal) $$D_\text{KL}(\mathcal{N}(\mu, \sigma^2)\,\|\,\mathcal{N}(0,1)) = \int \mathcal{N}(z;\mu,\sigma^2)\log\frac{\mathcal{N}(z;\mu,\sigma^2)}{\mathcal{N}(z;0,1)}\,dz$$ $$= \int \mathcal{N}(z;\mu,\sigma^2)\!\left[-\frac{(z-\mu)^2}{2\sigma^2} + \frac{z^2}{2} + \frac{1}{2}\log\sigma^2\right]\!dz$$ $$= -\frac{1}{2} + \frac{\mu^2 + \sigma^2}{2} + \frac{1}{2}\log\sigma^2$$

Usando \(\mathbb{E}[Z^2] = \mu^2 + \sigma^2\) y la entropía de una gaussiana. Para \(d_z\) dimensiones independientes, se suma sobre \(j\): \(= -\frac{1}{2}\sum_j(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2)\;\blacksquare\)

Este resultado es crucial: el término KL del ELBO se computa exactamente, y solo el término de reconstrucción requiere estimación Monte Carlo (con \(S=1\) muestra vía reparametrización).

$$\mathcal{L}(\theta, \phi) = \sum_{i=1}^N \mathcal{L}(\theta, \phi;\, \mathbf{x}^{(i)})$$

1. El modelo VAE completo

vae_model.py Python · PyTorch 
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import Tuple


class Encoder(nn.Module):
    """
    Red de inferencia: q_φ(z|x)
    
    Mapea x -> (μ, log σ²) — los parámetros del posterior aproximado.
    Usamos log σ² (no σ) para que el output sea sin restricciones.
    """
    def __init__(self, input_dim: int, hidden_dim: int, latent_dim: int):
        super().__init__()
        
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
        )
        
        # Dos cabezas: una para μ, otra para log σ²
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)
    
    def forward(self, x: torch.Tensor) -> Tuple[torch.Tensor, torch.Tensor]:
        """
        Returns:
            mu:     (B, latent_dim) — media del posterior
            logvar: (B, latent_dim) — log varianza del posterior (= log σ²)
        """
        h = self.net(x)
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)  # sin restricción: puede ser cualquier real
        return mu, logvar


class Decoder(nn.Module):
    """
    Red generativa: p_θ(x|z)
    
    Para MNIST usamos Bernoulli likelihood:
        p_θ(x|z) = Bernoulli(π_θ(z))
    -> Loss = Binary Cross Entropy
    
    Para datos continuos se usaría:
        p_θ(x|z) = N(μ_θ(z), σ²I)
    -> Loss = MSE (o equivalentemente Gaussian NLL)
    """
    def __init__(self, latent_dim: int, hidden_dim: int, output_dim: int):
        super().__init__()
        
        self.net = nn.Sequential(
            nn.Linear(latent_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, output_dim),
            # Sin activación final — usaremos BCEWithLogitsLoss
        )
    
    def forward(self, z: torch.Tensor) -> torch.Tensor:
        """
        Returns:
            logits: (B, output_dim) — logits de la Bernoulli (pre-sigmoid)
        """
        return self.net(z)


class VAE(nn.Module):
    """
    Variational Autoencoder completo.
    
    Arquitectura:
        x -> Encoder -> (μ, log σ²) -> z = μ + σ·ε -> Decoder -> x̂
    
    Objetivo (ELBO):
        L(θ, φ; x) = E_q[log p_θ(x|z)] - KL(q_φ(z|x) || N(0,I))
                   = -Recon_Loss          + KL_Loss (negativo, se suma)
    """
    def __init__(
        self,
        input_dim: int = 784,    # 28*28 para MNIST
        hidden_dim: int = 512,
        latent_dim: int = 20,
    ):
        super().__init__()
        self.latent_dim = latent_dim
        self.encoder = Encoder(input_dim, hidden_dim, latent_dim)
        self.decoder = Decoder(latent_dim, hidden_dim, input_dim)
    
    def reparametrize(
        self, mu: torch.Tensor, logvar: torch.Tensor
    ) -> torch.Tensor:
        """
        Truco de reparametrización:
            z = μ + σ · ε,   ε ~ N(0, I)
        
        Durante inferencia (eval mode) retornamos simplemente μ.
        """
        if self.training:
            std = torch.exp(0.5 * logvar)   # σ = exp(log σ) = exp(0.5 · log σ²)
            eps = torch.randn_like(std)      # ε ~ N(0, I), sin gradiente
            return mu + std * eps            # z — diferenciable respecto a μ, logvar
        else:
            return mu  # máxima verosimilitud en eval
    
    def forward(
        self, x: torch.Tensor
    ) -> Tuple[torch.Tensor, torch.Tensor, torch.Tensor]:
        """
        Forward completo del VAE.
        
        Returns:
            recon_logits: (B, input_dim) — logits de reconstrucción
            mu:           (B, latent_dim) — media del posterior
            logvar:       (B, latent_dim) — log varianza del posterior
        """
        # Codificar: x -> (μ, log σ²)
        mu, logvar = self.encoder(x)
        
        # Reparametrizar: muestrear z ~ q_φ(z|x)
        z = self.reparametrize(mu, logvar)
        
        # Decodificar: z -> x̂ (logits)
        recon_logits = self.decoder(z)
        
        return recon_logits, mu, logvar
    
    @torch.no_grad()
    def generate(self, n_samples: int, device: str = 'cpu') -> torch.Tensor:
        """
        Generación: muestrea z ~ p(z) = N(0,I) y decodifica.
        Este es el proceso generativo del modelo.
        """
        z = torch.randn(n_samples, self.latent_dim, device=device)
        logits = self.decoder(z)
        return torch.sigmoid(logits)  # probabilidades Bernoulli
    
    @torch.no_grad()
    def reconstruct(self, x: torch.Tensor) -> torch.Tensor:
        """Reconstruye usando la media del posterior (sin ruido)."""
        self.eval()
        mu, _ = self.encoder(x)
        logits = self.decoder(mu)
        return torch.sigmoid(logits)

2. La función de pérdida — ELBO

vae_loss.py Python · PyTorch 
import torch
import torch.nn.functional as F


def vae_elbo_loss(
    recon_logits: torch.Tensor,   # (B, D) — output del decoder (pre-sigmoid)
    x: torch.Tensor,              # (B, D) — input original
    mu: torch.Tensor,             # (B, latent_dim)
    logvar: torch.Tensor,         # (B, latent_dim)
    beta: float = 1.0,            # peso del término KL (β-VAE si β > 1)
    reduction: str = 'mean'       # 'mean' o 'sum'
) -> dict:
    """
    Computa el ELBO del VAE:
    
        ELBO = E_q[log p(x|z)] - β · KL(q_φ(z|x) || N(0,I))
    
    Implementación:
        - Reconstrucción: Binary Cross Entropy (Bernoulli likelihood)
        - KL: forma cerrada analítica
    
    Returns:
        dict con 'loss' (negativo ELBO), 'recon_loss', 'kl_loss'
    """
    B = x.shape[0]
    
    # ── Término de reconstrucción ──────────────────────────────────────────
    # E_q[log p_θ(x|z)] ≈ -BCE(sigmoid(recon_logits), x)
    # 
    # BCEWithLogitsLoss es numéricamente más estable que 
    # sigmoid + BCELoss, y es equivalente.
    # reduction='sum' divide por batch size después para consistencia.
    recon_loss = F.binary_cross_entropy_with_logits(
        recon_logits, x, reduction='sum'
    ) / B  # normalizar por batch
    
    # ── Término KL (forma cerrada) ─────────────────────────────────────────
    # KL(N(μ, σ²) || N(0, 1)) = -½ · Σ_j (1 + log σ²_j - μ²_j - σ²_j)
    #
    # Nota: logvar = log(σ²), por tanto σ² = exp(logvar)
    kl_per_dim = 1 + logvar - mu.pow(2) - logvar.exp()  # (B, latent_dim)
    kl_loss = -0.5 * kl_per_dim.sum(dim=1).mean()       # escalar
    
    # ── ELBO total (negativo para minimizar) ──────────────────────────────
    # Minimizamos -ELBO = recon_loss + β · kl_loss
    loss = recon_loss + beta * kl_loss
    
    return {
        'loss': loss,
        'recon_loss': recon_loss,
        'kl_loss': kl_loss,
        'elbo': -(recon_loss + kl_loss),  # ELBO verdadero (para monitorear)
    }


def kl_analytical(mu: torch.Tensor, logvar: torch.Tensor) -> torch.Tensor:
    """
    KL-divergencia analítica, versión standalone.
    
    KL(N(μ,σ²) || N(0,1)) = -½ Σ_j (1 + log σ²_j - μ²_j - σ²_j)
    
    Valores de referencia:
        μ=0, σ²=1 → KL = 0        (posterior = prior, sin información)
        μ=0, σ²→0 → KL → ∞        (posterior colapsa, overfitting)
        |μ|→∞    → KL → ∞        (posterior lejos del prior)
    """
    return -0.5 * (1 + logvar - mu.pow(2) - logvar.exp()).sum(dim=-1)


def gaussian_nll_loss(
    x: torch.Tensor,
    mu_recon: torch.Tensor,
    log_var_recon: float = 0.0   # log σ² del decoder (fijo = 0 → σ²=1)
) -> torch.Tensor:
    """
    Negative log-likelihood gaussiana para datos continuos.
    
    p_θ(x|z) = N(μ_θ(z), σ²I)
    -log p_θ(x|z) = ½ log(2πσ²) + ||x - μ||² / (2σ²)
    ∝ MSE(x, μ_recon)   (cuando σ² es constante)
    """
    sigma_sq = torch.exp(torch.tensor(log_var_recon))
    nll = 0.5 * (
        torch.log(2 * torch.pi * sigma_sq) 
        + (x - mu_recon).pow(2) / sigma_sq
    )
    return nll.sum(dim=-1).mean()

import torch
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import matplotlib.pyplot as plt
from tqdm import tqdm


def train_vae(
    latent_dim: int = 20,
    hidden_dim: int = 512,
    epochs: int = 30,
    batch_size: int = 128,
    lr: float = 1e-3,
    beta: float = 1.0,
    device: str = 'cuda' if torch.cuda.is_available() else 'cpu'
):
    print(f"Training VAE | latent_dim={latent_dim} | β={beta} | device={device}")
    
    # ── Dataset ──────────────────────────────────────────────────────────
    transform = transforms.Compose([
        transforms.ToTensor(),
        transforms.Lambda(lambda x: x.view(-1))  # aplanar a (784,)
    ])
    train_data = datasets.MNIST('.', train=True, download=True, transform=transform)
    val_data   = datasets.MNIST('.', train=False, download=True, transform=transform)
    
    train_loader = DataLoader(train_data, batch_size=batch_size, shuffle=True,
                              num_workers=4, pin_memory=True)
    val_loader   = DataLoader(val_data, batch_size=batch_size, shuffle=False)
    
    # ── Modelo y optimizador ─────────────────────────────────────────────
    model = VAE(input_dim=784, hidden_dim=hidden_dim, latent_dim=latent_dim)
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr)
    
    history = {'train_loss': [], 'val_elbo': [], 'kl': [], 'recon': []}
    
    for epoch in range(epochs):
        # ── Training ────────────────────────────────────────────────────
        model.train()
        train_losses = []
        
        for x, _ in tqdm(train_loader, desc=f'Epoch {epoch+1}/{epochs}', 
                          leave=False):
            x = x.to(device)
            optimizer.zero_grad()
            
            # Forward: x -> (recon_logits, μ, log σ²) -> z
            recon_logits, mu, logvar = model(x)
            
            # Computar ELBO
            losses = vae_elbo_loss(recon_logits, x, mu, logvar, beta=beta)
            
            # Backprop — gradientes fluyen a través de reparametrización
            losses['loss'].backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()
            
            train_losses.append({k: v.item() for k, v in losses.items()})
        
        # ── Validation ─────────────────────────────────────────────────
        model.eval()
        val_elbos = []
        with torch.no_grad():
            for x, _ in val_loader:
                x = x.to(device)
                recon_logits, mu, logvar = model(x)
                losses = vae_elbo_loss(recon_logits, x, mu, logvar)
                val_elbos.append(losses['elbo'].item())
        
        # ── Logging ────────────────────────────────────────────────────
        avg_train = sum(d['loss'] for d in train_losses) / len(train_losses)
        avg_kl    = sum(d['kl_loss'] for d in train_losses) / len(train_losses)
        avg_recon = sum(d['recon_loss'] for d in train_losses) / len(train_losses)
        avg_val   = sum(val_elbos) / len(val_elbos)
        
        history['train_loss'].append(avg_train)
        history['val_elbo'].append(avg_val)
        history['kl'].append(avg_kl)
        history['recon'].append(avg_recon)
        
        print(f"Epoch {epoch+1:3d} | "
              f"Loss={avg_train:.2f} | "
              f"Recon={avg_recon:.2f} | "
              f"KL={avg_kl:.2f} | "
              f"Val ELBO={avg_val:.2f}")
    
    return model, history

@torch.no_grad()
def plot_reconstructions(model, data_loader, device, n=8):

    axes[1, 0].set_ylabel('Reconstrucción', fontsize=10)
    plt.tight_layout()
    plt.show()


@torch.no_grad()
def plot_latent_space(model, data_loader, device, n_points=2000):
    """
    Visualiza el espacio latente 2D (solo funciona bien con latent_dim=2).
    Muestra cómo el prior N(0,I) organiza las clases.
    """
    model.eval()
    mus, labels = [], []
    
    for x, y in data_loader:
        mu, _ = model.encoder(x.to(device))
        mus.append(mu.cpu())
        labels.append(y)
        if sum(len(m) for m in mus) >= n_points:
            break
    
    mus = torch.cat(mus)[:n_points]
    labels = torch.cat(labels)[:n_points]
    
    plt.figure(figsize=(8, 6))
    scatter = plt.scatter(mus[:, 0], mus[:, 1], c=labels, 
                           cmap='tab10', alpha=0.5, s=8)
    plt.colorbar(scatter)
    plt.title('Espacio latente 2D — medias del posterior q_φ(z|x)')
    plt.xlabel('z₁'); plt.ylabel('z₂')
    
    # Mostrar el prior N(0,I) como círculo
    theta = torch.linspace(0, 2*3.14159, 100)
    plt.plot(torch.cos(theta), torch.sin(theta), 'k--', 
             alpha=0.3, label='1σ prior')
    plt.plot(2*torch.cos(theta), 2*torch.sin(theta), 'k:', 
             alpha=0.2, label='2σ prior')
    plt.legend()
    plt.tight_layout()
    plt.show()


@torch.no_grad()
def interpolate_latents(model, x1, x2, device, n_steps=10):
    """
    Interpolación lineal entre dos imágenes en el espacio latente.
    Demuestra la suavidad del espacio latente aprendido.
    """
    model.eval()
    x1, x2 = x1.to(device), x2.to(device)
    
    mu1, _ = model.encoder(x1.unsqueeze(0))
    mu2, _ = model.encoder(x2.unsqueeze(0))
    
    alphas = torch.linspace(0, 1, n_steps)
    
    fig, axes = plt.subplots(1, n_steps, figsize=(n_steps * 1.5, 2))
    for i, alpha in enumerate(alphas):
        z_interp = (1 - alpha) * mu1 + alpha * mu2
        recon = torch.sigmoid(model.decoder(z_interp))
        axes[i].imshow(recon.cpu().view(28, 28), cmap='gray')
        axes[i].axis('off')
        axes[i].set_title(f'{alpha:.1f}', fontsize=8)
    
    plt.suptitle('Interpolación en espacio latente', fontsize=11)
    plt.tight_layout()
    plt.show()


# ── Verificación del ELBO y sus componentes ──────────────────────────────────

def analyze_elbo_components(model, data_loader, device, n_batches=10):
    """
    Análisis cuantitativo del ELBO y sus componentes.
    Verifica que ELBO = Recon + KL, y monitorea el balance.
    """
    model.eval()
    stats = {'elbo': [], 'recon': [], 'kl': [], 'kl_per_dim': []}
    
    with torch.no_grad():
        for i, (x, _) in enumerate(data_loader):
            if i >= n_batches:
                break
            x = x.to(device)
            recon_logits, mu, logvar = model(x)
            losses = vae_elbo_loss(recon_logits, x, mu, logvar)
            
            stats['elbo'].append(losses['elbo'].item())
            stats['recon'].append(losses['recon_loss'].item())
            stats['kl'].append(losses['kl_loss'].item())
            # KL por dimensión latente (identifica dims informativas)
            kl_dims = kl_analytical(mu, logvar).mean(dim=0)  # per-sample average
            stats['kl_per_dim'].append(kl_dims.cpu())
    
    print("\n" + "="*50)
    print("ELBO Components Analysis")
    print("="*50)
    print(f"ELBO (↑ mejor): {sum(stats['elbo'])/len(stats['elbo']):.3f}")
    print(f"Recon loss (↓): {sum(stats['recon'])/len(stats['recon']):.3f}")
    print(f"KL loss   (↓):  {sum(stats['kl'])/len(stats['kl']):.3f}")
    
    # KL por dimensión — dimensiones con KL≈0 son "inactivas" (posterior collapse)
    kl_per_dim = torch.stack(stats['kl_per_dim']).mean(dim=0)
    n_active = (kl_per_dim > 0.1).sum().item()
    print(f"\nDimensiones latentes activas: {n_active}/{len(kl_per_dim)}")
    print(f"KL promedio por dim: {kl_per_dim.mean():.3f}")
    print(f"Min KL por dim: {kl_per_dim.min():.4f}")
    print(f"Max KL por dim: {kl_per_dim.max():.3f}")
    
    return stats

conv_vae.py Python · PyTorch — ConvVAE para imágenes 
import torch
import torch.nn as nn


class ConvEncoder(nn.Module):
    """
    Encoder convolucional: x (B,C,H,W) -> (μ, log σ²)
    Apropiado para imágenes de mayor resolución.
    """
    def __init__(self, in_channels: int = 1, latent_dim: int = 128):
        super().__init__()
        
        # Extractor convolucional de features
        self.conv = nn.Sequential(
            # (B, 1, 28, 28) -> (B, 32, 14, 14)
            nn.Conv2d(in_channels, 32, 4, stride=2, padding=1),
            nn.BatchNorm2d(32), nn.LeakyReLU(0.2),
            
            # -> (B, 64, 7, 7)
            nn.Conv2d(32, 64, 4, stride=2, padding=1),
            nn.BatchNorm2d(64), nn.LeakyReLU(0.2),
            
            # -> (B, 128, 4, 4) (con padding ajustado)
            nn.Conv2d(64, 128, 3, stride=1, padding=1),
            nn.BatchNorm2d(128), nn.LeakyReLU(0.2),
        )
        
        # Dimensión del feature map aplanado: 128 * 7 * 7
        self.flat_dim = 128 * 7 * 7
        
        self.fc_mu     = nn.Linear(self.flat_dim, latent_dim)
        self.fc_logvar = nn.Linear(self.flat_dim, latent_dim)
    
    def forward(self, x):
        h = self.conv(x).flatten(start_dim=1)   # (B, flat_dim)
        return self.fc_mu(h), self.fc_logvar(h)


class ConvDecoder(nn.Module):
    """
    Decoder convolucional: z -> x̂ (B,C,H,W)
    Usa ConvTranspose2d para "upsampling aprendido".
    """
    def __init__(self, latent_dim: int = 128, out_channels: int = 1):
        super().__init__()
        
        self.flat_dim = 128 * 7 * 7
        self.fc = nn.Linear(latent_dim, self.flat_dim)
        
        self.deconv = nn.Sequential(
            nn.BatchNorm2d(128), nn.LeakyReLU(0.2),
            
            # (B, 128, 7, 7) -> (B, 64, 7, 7)
            nn.ConvTranspose2d(128, 64, 3, stride=1, padding=1),
            nn.BatchNorm2d(64), nn.LeakyReLU(0.2),
            
            # -> (B, 32, 14, 14)
            nn.ConvTranspose2d(64, 32, 4, stride=2, padding=1),
            nn.BatchNorm2d(32), nn.LeakyReLU(0.2),
            
            # -> (B, 1, 28, 28)
            nn.ConvTranspose2d(32, out_channels, 4, stride=2, padding=1),
            # Sin activación: usamos BCEWithLogitsLoss
        )
    
    def forward(self, z):
        h = self.fc(z).view(-1, 128, 7, 7)
        return self.deconv(h)


class ConvVAE(nn.Module):
    """VAE completamente convolucional."""
    
    def __init__(self, in_channels=1, latent_dim=128):
        super().__init__()
        self.latent_dim = latent_dim
        self.encoder = ConvEncoder(in_channels, latent_dim)
        self.decoder = ConvDecoder(latent_dim, in_channels)
    
    def reparametrize(self, mu, logvar):
        if self.training:
            return mu + torch.exp(0.5 * logvar) * torch.randn_like(mu)
        return mu
    
    def forward(self, x):
        mu, logvar = self.encoder(x)
        z = self.reparametrize(mu, logvar)
        return self.decoder(z), mu, logvar
    
    @torch.no_grad()
    def generate(self, n, device='cpu'):
        z = torch.randn(n, self.latent_dim, device=device)
        return torch.sigmoid(self.decoder(z))
