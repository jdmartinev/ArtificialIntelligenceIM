§ 00 — Motivación

¿Por qué difusión?

Los modelos generativos buscan aprender una distribución \(p_\theta(\mathbf{x})\) que aproxime la distribución real de los datos \(p^*(\mathbf{x})\). Contamos con varias familias:

Familia Fortaleza Debilidad central
GANs Calidad visual alta Inestabilidad de entrenamiento, mode collapse
VAEs ELBO tractable, latentes interpretables Imágenes borrosas (posterior collapse)
Flow models Likelihood exacta Arquitecturas restrictivas (bijecciones)
Diffusion Alta calidad, entrenamiento estable, ELBO tractable Muestreo lento (T pasos)

La idea central de los modelos de difusión es elegante: destruir los datos gradualmente con ruido gaussiano (proceso forward), y después entrenar una red neuronal para invertir ese proceso (proceso reverse). La clave matemática: el proceso de destrucción es suficientemente simple como para tener formas cerradas en todo momento.

Imagina que tenemos una imagen de un perro. Si añadimos ruido gaussiano en T = 1000 pasos pequeños, eventualmente obtenemos ruido puro \(\mathcal{N}(0, I)\). Si entrenamos una red para deshacer cada paso de ruido, podemos generar nuevas imágenes partiendo de ruido puro.
§ 01 — Forward Process

El proceso forward

Definimos el proceso forward como una cadena de Markov que añade ruido gaussiano con varianza \(\beta_t\) en cada paso \(t = 1, \dots, T\):

Definición — Forward Process $$q(\mathbf{x}_{1:T} \mid \mathbf{x}_0) := \prod_{t=1}^{T} q(\mathbf{x}_t \mid \mathbf{x}_{t-1})$$ $$q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) := \mathcal{N}\!\left(\mathbf{x}_t;\; \sqrt{1-\beta_t}\,\mathbf{x}_{t-1},\; \beta_t \mathbf{I}\right)$$

Aquí \(\{\beta_t\}_{t=1}^T\) es un noise schedule fijo (no aprendido), con \(\beta_t \in (0,1)\). Típicamente se usa un schedule lineal: \(\beta_1 = 10^{-4}, \dots, \beta_T = 0.02\) con \(T=1000\).

El truco clave: forma cerrada en cualquier \(t\)

Una propiedad fundamental es que podemos obtener \(\mathbf{x}_t\) directamente desde \(\mathbf{x}_0\) sin iterar los \(t\) pasos. Definimos:

Definición $$\alpha_t := 1 - \beta_t \qquad \bar{\alpha}_t := \prod_{s=1}^{t} \alpha_s$$

Entonces, usando la propiedad de que la suma de gaussianas es gaussiana, se puede demostrar por inducción:

Resultado clave — Marginal del forward process $$q(\mathbf{x}_t \mid \mathbf{x}_0) = \mathcal{N}\!\left(\mathbf{x}_t;\; \sqrt{\bar{\alpha}_t}\,\mathbf{x}_0,\; (1-\bar{\alpha}_t)\mathbf{I}\right)$$

Equivalentemente, usando el truco de reparametrización:

$$\mathbf{x}_t = \sqrt{\bar{\alpha}_t}\,\mathbf{x}_0 + \sqrt{1 - \bar{\alpha}_t}\,\boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(0, \mathbf{I})$$
¿Por qué esto importa? Durante el entrenamiento necesitamos muestras de \(\mathbf{x}_t\) para cualquier \(t\) arbitrario. Sin esta forma cerrada, tendríamos que simular toda la cadena desde \(\mathbf{x}_0\) hasta \(\mathbf{x}_t\) — ineficiente. Con esta fórmula, podemos ir directamente al paso \(t\) en una sola operación.

Derivación del marginal

Derivación por inducción

Caso base \(t=1\): \(\mathbf{x}_1 = \sqrt{\alpha_1}\mathbf{x}_0 + \sqrt{1-\alpha_1}\boldsymbol{\epsilon}_1\).

Paso inductivo: asumiendo \(\mathbf{x}_{t-1} = \sqrt{\bar{\alpha}_{t-1}}\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_{t-1}}\boldsymbol{\epsilon}_{t-1}\), entonces:

$$\mathbf{x}_t = \sqrt{\alpha_t}\mathbf{x}_{t-1} + \sqrt{1-\alpha_t}\boldsymbol{\epsilon}_t$$ $$= \sqrt{\alpha_t}\left(\sqrt{\bar{\alpha}_{t-1}}\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_{t-1}}\boldsymbol{\epsilon}_{t-1}\right) + \sqrt{1-\alpha_t}\boldsymbol{\epsilon}_t$$ $$= \sqrt{\bar{\alpha}_t}\mathbf{x}_0 + \underbrace{\sqrt{\alpha_t(1-\bar{\alpha}_{t-1})}\boldsymbol{\epsilon}_{t-1} + \sqrt{1-\alpha_t}\boldsymbol{\epsilon}_t}_{\sim\,\mathcal{N}(0,\,(1-\bar\alpha_t)\mathbf{I})}$$

La última suma es gaussiana con varianza \(\alpha_t(1-\bar\alpha_{t-1}) + (1-\alpha_t) = 1 - \alpha_t\bar\alpha_{t-1} = 1 - \bar\alpha_t\). \(\blacksquare\)

Cuando \(T \to \infty\) (o \(T\) es suficientemente grande), \(\bar{\alpha}_T \approx 0\) y por tanto \(q(\mathbf{x}_T \mid \mathbf{x}_0) \approx \mathcal{N}(0, \mathbf{I})\). El dato original queda completamente destruido.

§ 02 — Reverse Process

El proceso reverse

Si el proceso forward destruye datos, el proceso reverse los genera. Empezamos desde ruido puro \(\mathbf{x}_T \sim \mathcal{N}(0, \mathbf{I})\) y queremos recuperar \(\mathbf{x}_0\).

El reverse verdadero: \(q(\mathbf{x}_{t-1} \mid \mathbf{x}_t)\)

Por el teorema de Bayes, el verdadero proceso reverso es:

$$q(\mathbf{x}_{t-1} \mid \mathbf{x}_t) = \frac{q(\mathbf{x}_t \mid \mathbf{x}_{t-1})\,q(\mathbf{x}_{t-1})}{q(\mathbf{x}_t)}$$
Para calcular \(q(\mathbf{x}_{t-1} \mid \mathbf{x}_t)\) necesitamos \(q(\mathbf{x}_{t-1})\) y \(q(\mathbf{x}_t)\) — las marginales del proceso forward. Estas requieren integrar sobre todos los posibles \(\mathbf{x}_0\), lo cual es intratable para distribuciones complejas de datos.

La solución: aproximar con una red neuronal

Aproximamos el reverse process con una distribución paramétrica:

Definición — Reverse parametrizado $$p_\theta(\mathbf{x}_{0:T}) := p(\mathbf{x}_T)\prod_{t=1}^{T} p_\theta(\mathbf{x}_{t-1} \mid \mathbf{x}_t)$$ $$p_\theta(\mathbf{x}_{t-1} \mid \mathbf{x}_t) := \mathcal{N}\!\left(\mathbf{x}_{t-1};\; \boldsymbol{\mu}_\theta(\mathbf{x}_t, t),\; \Sigma_\theta(\mathbf{x}_t, t)\right)$$

La red neuronal con parámetros \(\theta\) aprende a predecir la media (y opcionalmente la varianza) de la distribución del paso anterior dado el actual. En la práctica, Ho et al. fijan \(\Sigma_\theta = \sigma_t^2 \mathbf{I}\) con \(\sigma_t\) fijo.

El prior es simplemente: \(p(\mathbf{x}_T) = \mathcal{N}(0, \mathbf{I})\).

§ 03 — El Problema de la Intratabilidad

¿Por qué no maximizar la likelihood directamente?

Queremos encontrar \(\theta^* = \arg\max_\theta \log p_\theta(\mathbf{x}_0)\). Expandiendo:

$$\log p_\theta(\mathbf{x}_0) = \log \int p_\theta(\mathbf{x}_{0:T})\, d\mathbf{x}_{1:T}$$

Esta integral es sobre un espacio de dimensión \(T \times d\) donde \(d\) es la dimensión de los datos. Para imágenes \(256 \times 256\) con \(T=1000\), integramos sobre un espacio de dimensión \(\sim 2 \times 10^8\). Completamente intratable.

Tampoco podemos estimar el gradiente por Monte Carlo directamente porque la integral involucra \(p_\theta\), no una distribución fija.

Este es exactamente el mismo problema que en los VAEs. La solución es la misma: introducir una distribución variacional \(q\) y derivar una cota inferior tractable — el ELBO.
§ 04 — El ELBO

Evidence Lower Bound

Introducimos la distribución forward \(q(\mathbf{x}_{1:T} \mid \mathbf{x}_0)\) como distribución variacional. Aplicamos la desigualdad de Jensen al logaritmo (función cóncava):

Derivación del ELBO $$\log p_\theta(\mathbf{x}_0) = \log \int p_\theta(\mathbf{x}_{0:T})\, d\mathbf{x}_{1:T}$$ $$= \log \int \frac{p_\theta(\mathbf{x}_{0:T})}{q(\mathbf{x}_{1:T}|\mathbf{x}_0)} \cdot q(\mathbf{x}_{1:T}|\mathbf{x}_0)\, d\mathbf{x}_{1:T}$$ $$= \log \,\mathbb{E}_{q(\mathbf{x}_{1:T}|\mathbf{x}_0)}\!\left[\frac{p_\theta(\mathbf{x}_{0:T})}{q(\mathbf{x}_{1:T}|\mathbf{x}_0)}\right]$$ $$\geq \mathbb{E}_{q}\!\left[\log \frac{p_\theta(\mathbf{x}_{0:T})}{q(\mathbf{x}_{1:T}|\mathbf{x}_0)}\right] \quad \leftarrow \textit{Jensen's inequality}$$ $$=: \mathcal{L}_\text{ELBO}$$

La brecha entre la log-likelihood y el ELBO es exactamente:

$$\log p_\theta(\mathbf{x}_0) - \mathcal{L}_\text{ELBO} = D_\text{KL}\!\left(q(\mathbf{x}_{1:T}|\mathbf{x}_0)\;\|\;p_\theta(\mathbf{x}_{1:T}|\mathbf{x}_0)\right) \geq 0$$

Como la KL-divergencia es siempre no-negativa, maximizar el ELBO implica maximizar una cota inferior de la log-likelihood. Cuando el ELBO es ajustado, la distribución variacional \(q\) aproxima bien el posterior verdadero.

Por qué el ELBO es ahora tractable

El ELBO involucra una esperanza respecto a \(q\), que sí podemos estimar por Monte Carlo — simplemente muestreamos \(\mathbf{x}_{1:T} \sim q(\cdot|\mathbf{x}_0)\) usando el truco de reparametrización.

§ 05 — Los Tres Términos del ELBO

Descomposición y significado de cada término

Expandiendo el ELBO usando las definiciones de \(p_\theta\) y \(q\) como cadenas de Markov, y tomando logaritmos:

Expansión completa $$\mathcal{L}_\text{ELBO} = \mathbb{E}_q\left[\log\frac{p(\mathbf{x}_T)\prod_{t=1}^T p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t)}{\prod_{t=1}^T q(\mathbf{x}_t|\mathbf{x}_{t-1})}\right]$$ $$= \mathbb{E}_q\left[\log p_\theta(\mathbf{x}_0|\mathbf{x}_1) + \sum_{t=2}^T \log\frac{p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t)}{q(\mathbf{x}_t|\mathbf{x}_{t-1})} + \log\frac{p(\mathbf{x}_T)}{q(\mathbf{x}_T|\mathbf{x}_{T-1})}\right]$$

Después de manipulaciones algebráicas (reescribir \(q(\mathbf{x}_t|\mathbf{x}_{t-1})\) usando el posterior condicionado en \(\mathbf{x}_0\)), obtenemos la descomposición limpia:

Descomposición final del ELBO $$-\mathcal{L}_\text{ELBO} = \underbrace{D_\text{KL}(q(\mathbf{x}_T|\mathbf{x}_0)\,\|\,p(\mathbf{x}_T))}_{\mathcal{L}_T} + \sum_{t=2}^{T}\underbrace{D_\text{KL}(q(\mathbf{x}_{t-1}|\mathbf{x}_t,\mathbf{x}_0)\,\|\,p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t))}_{\mathcal{L}_{t-1}} - \underbrace{\mathbb{E}_q[\log p_\theta(\mathbf{x}_0|\mathbf{x}_1)]}_{\mathcal{L}_0}$$

Maximizamos el ELBO = minimizamos \(-\mathcal{L}_\text{ELBO}\). Veamos cada término:

\(\mathcal{L}_T\) — Prior Matching Term
$$D_\text{KL}(q(\mathbf{x}_T|\mathbf{x}_0)\,\|\,p(\mathbf{x}_T))$$

¿Qué mide? La distancia entre la distribución del dato original después de \(T\) pasos de ruido, y el prior gaussiano puro.

¿Por qué no importa? Si el noise schedule está bien calibrado, para \(T\) suficientemente grande \(q(\mathbf{x}_T|\mathbf{x}_0) \approx \mathcal{N}(0,\mathbf{I}) = p(\mathbf{x}_T)\), haciendo este término \(\approx 0\). No tiene parámetros \(\theta\) — no contribuye al gradiente. Se ignora en entrenamiento.

\(\mathcal{L}_{t-1}\) — Denoising Matching Terms (los importantes)
$$D_\text{KL}(q(\mathbf{x}_{t-1}|\mathbf{x}_t,\mathbf{x}_0)\,\|\,p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t)), \quad t = 2,\dots,T$$

¿Qué mide? En cada paso \(t\), compara dos distribuciones:

— \(q(\mathbf{x}_{t-1}|\mathbf{x}_t, \mathbf{x}_0)\): el denoising step óptimo (condicionado en \(\mathbf{x}_0\), que no conocemos en generación)

— \(p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t)\): lo que predice nuestra red neuronal (sin ver \(\mathbf{x}_0\))

Intuición: Le decimos a la red: "en cada paso, tu distribución predicha debe coincidir con la distribución ideal que conoce la verdad". Minimizar esta KL es el corazón del entrenamiento.

¿Por qué es KL entre gaussianas? ¡Porque \(q(\mathbf{x}_{t-1}|\mathbf{x}_t, \mathbf{x}_0)\) es gaussiana con forma cerrada! (Ver §06)

\(\mathcal{L}_0\) — Reconstruction Term
$$\mathbb{E}_q[\log p_\theta(\mathbf{x}_0|\mathbf{x}_1)]$$

¿Qué mide? La verosimilitud de reconstruir el dato original \(\mathbf{x}_0\) a partir de la versión levemente ruidosa \(\mathbf{x}_1\).

Rol: Corresponde al paso final de generación — pasar de \(\mathbf{x}_1\) (casi sin ruido) a la imagen final. En Ho et al. se trata de forma especial discretizando los valores de píxel.

Analogía con VAE: Este término es el equivalente exacto al término de reconstrucción \(\mathbb{E}[\log p_\theta(\mathbf{x}|\mathbf{z})]\) en los VAEs.

Resumen de los tres términos

\(\mathcal{L}_T\) — "¿Destruimos suficiente?" → Casi siempre \(\approx 0\), no se optimiza.

\(\mathcal{L}_{t-1}\) — "¿Sabe la red deshacer el ruido en cada paso?" → El corazón del entrenamiento. Son \(T-1\) términos KL entre gaussianas.

\(\mathcal{L}_0\) — "¿Puede reconstruir el dato final?" → Paso de reconstrucción, comparable al VAE.

§ 06 — El Posterior Forward es Tratable

Forma cerrada de \(q(\mathbf{x}_{t-1}|\mathbf{x}_t, \mathbf{x}_0)\)

Esta es la pieza que hace que todo funcione. El posterior del forward process condicionado en \(\mathbf{x}_0\) tiene una forma gaussiana cerrada.

Derivación por Bayes $$q(\mathbf{x}_{t-1}|\mathbf{x}_t, \mathbf{x}_0) = \frac{q(\mathbf{x}_t|\mathbf{x}_{t-1})\,q(\mathbf{x}_{t-1}|\mathbf{x}_0)}{q(\mathbf{x}_t|\mathbf{x}_0)}$$

Los tres factores son gaussianas conocidas. Tomando logaritmos y completando el cuadrado:

Resultado — Posterior Forward $$q(\mathbf{x}_{t-1}|\mathbf{x}_t, \mathbf{x}_0) = \mathcal{N}\!\left(\mathbf{x}_{t-1};\; \tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t, \mathbf{x}_0),\; \tilde{\beta}_t \mathbf{I}\right)$$ $$\text{donde:} \quad \tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t, \mathbf{x}_0) = \frac{\sqrt{\bar{\alpha}_{t-1}}\beta_t}{1-\bar{\alpha}_t}\mathbf{x}_0 + \frac{\sqrt{\alpha_t}(1-\bar{\alpha}_{t-1})}{1-\bar{\alpha}_t}\mathbf{x}_t$$ $$\tilde{\beta}_t = \frac{1-\bar{\alpha}_{t-1}}{1-\bar{\alpha}_t}\cdot\beta_t$$

Esta media es una combinación lineal de \(\mathbf{x}_0\) y \(\mathbf{x}_t\): el "denoising step ideal" promedia entre la imagen original y la imagen ruidosa actual.

Conexión con la predicción de ruido

Usando que \(\mathbf{x}_0 = \frac{1}{\sqrt{\bar\alpha_t}}(\mathbf{x}_t - \sqrt{1-\bar\alpha_t}\,\boldsymbol{\epsilon})\), podemos reescribir \(\tilde{\boldsymbol{\mu}}_t\) en función del ruido \(\boldsymbol{\epsilon}\):

Parametrización alternativa $$\tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t, \boldsymbol{\epsilon}) = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}}\,\boldsymbol{\epsilon}\right)$$

Esta forma sugiere que la red neuronal debería parametrizarse como \(p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t) = \mathcal{N}(\mathbf{x}_{t-1};\,\boldsymbol{\mu}_\theta(\mathbf{x}_t, t), \tilde\beta_t\mathbf{I})\) con:

$$\boldsymbol{\mu}_\theta(\mathbf{x}_t, t) = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}}\,\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\right)$$

La red aprende a predecir el ruido \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t) \approx \boldsymbol{\epsilon}\) que se añadió en el paso \(t\).

§ 07 — Lo que se Optimiza en Práctica

De la KL-divergencia a la MSE

Dado que \(q(\mathbf{x}_{t-1}|\mathbf{x}_t,\mathbf{x}_0)\) y \(p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t)\) son ambas gaussianas con la misma varianza \(\tilde\beta_t \mathbf{I}\), la KL-divergencia entre ellas se reduce a:

KL entre gaussianas con varianza igual $$D_\text{KL}(\mathcal{N}(\boldsymbol{\mu}_1, \sigma^2\mathbf{I})\,\|\,\mathcal{N}(\boldsymbol{\mu}_2, \sigma^2\mathbf{I})) = \frac{1}{2\sigma^2}\|\boldsymbol{\mu}_1 - \boldsymbol{\mu}_2\|^2$$

Por lo tanto, minimizar \(\mathcal{L}_{t-1}\) equivale a minimizar:

$$\mathcal{L}_{t-1} \propto \mathbb{E}_{\mathbf{x}_0, \boldsymbol{\epsilon}}\!\left[\left\|\tilde{\boldsymbol{\mu}}_t - \boldsymbol{\mu}_\theta(\mathbf{x}_t, t)\right\|^2\right]$$

Sustituyendo la parametrización de \(\boldsymbol{\mu}_\theta\) en términos del predictor de ruido \(\boldsymbol{\epsilon}_\theta\):

Función de pérdida simplificada (Ho et al., 2020) $$\mathcal{L}_\text{simple} = \mathbb{E}_{t,\,\mathbf{x}_0,\,\boldsymbol{\epsilon}}\!\left[\left\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta\!\left(\sqrt{\bar{\alpha}_t}\mathbf{x}_0 + \sqrt{1-\bar\alpha_t}\boldsymbol{\epsilon},\; t\right)\right\|^2\right]$$

Ho et al. también proponen ignorar los factores de peso \(\frac{\beta_t^2}{2\tilde\beta_t\alpha_t(1-\bar\alpha_t)}\) que acompañan cada término \(\mathcal{L}_{t-1}\) (equivalente a un reweighting del ELBO), encontrando empíricamente que esto mejora la calidad de las muestras.

¿Qué hace la red neuronal? Recibe una imagen ruidosa \(\mathbf{x}_t\) en el paso \(t\) (codificado via embeddings de tiempo), y predice el ruido gaussiano \(\boldsymbol{\epsilon}\) que fue añadido. Es esencialmente un denoising autoencoder entrenado en todos los niveles de ruido simultáneamente.

Algoritmos completos

Training Sampling
1. Tomar \(\mathbf{x}_0 \sim q(\mathbf{x}_0)\)
2. Muestrear \(t \sim \text{Uniform}(1,T)\)
3. Muestrear \(\boldsymbol{\epsilon} \sim \mathcal{N}(0,\mathbf{I})\)
4. Calcular \(\mathbf{x}_t = \sqrt{\bar\alpha_t}\mathbf{x}_0 + \sqrt{1-\bar\alpha_t}\boldsymbol{\epsilon}\)
5. Gradiente: \(\nabla_\theta \|\boldsymbol\epsilon - \boldsymbol\epsilon_\theta(\mathbf{x}_t, t)\|^2\) 		1. Muestrear \(\mathbf{x}_T \sim \mathcal{N}(0,\mathbf{I})\)
2. Para \(t = T, T-1, \dots, 1\):
   a. Calcular \(\boldsymbol\mu_\theta(\mathbf{x}_t, t)\)
   b. Muestrear \(\mathbf{z} \sim \mathcal{N}(0,\mathbf{I})\) si \(t>1\)
   c. \(\mathbf{x}_{t-1} = \boldsymbol\mu_\theta + \sigma_t\mathbf{z}\)
3. Retornar \(\mathbf{x}_0\)

import torch
import torch.nn as nn
import numpy as np
import matplotlib.pyplot as plt
from typing import Tuple

# ── Noise Schedule ──────────────────────────────────────────────────────────

def linear_beta_schedule(T: int, beta_start: float = 1e-4, beta_end: float = 0.02):
    """
    Schedule lineal de Ho et al. (2020).
    beta_t crece linealmente de beta_start a beta_end.
    """
    return torch.linspace(beta_start, beta_end, T)

def cosine_beta_schedule(T: int, s: float = 0.008):
    """
    Schedule coseno de Nichol & Dhariwal (2021).
    Más suave en los extremos — mejora la calidad.
    
    ᾱ_t = cos²((t/T + s) / (1 + s) · π/2)
    """
    steps = torch.arange(T + 1, dtype=torch.float64) / T
    alphas_cumprod = torch.cos((steps + s) / (1 + s) * torch.pi / 2) ** 2
    alphas_cumprod = alphas_cumprod / alphas_cumprod[0]
    betas = 1 - alphas_cumprod[1:] / alphas_cumprod[:-1]
    return torch.clamp(betas, 0.0001, 0.9999).float()


# ── Precomputar coeficientes ──────────────────────────────────────────────────

class DiffusionConfig:
    """
    Precomputa todos los coeficientes necesarios para el forward 
    y reverse process dado un noise schedule.
    """
    def __init__(self, T: int = 1000, schedule: str = 'linear'):
        self.T = T
        
        if schedule == 'linear':
            self.betas = linear_beta_schedule(T)
        else:
            self.betas = cosine_beta_schedule(T)
        
        self.alphas = 1.0 - self.betas                         # α_t
        self.alphas_cumprod = torch.cumprod(self.alphas, dim=0) # ᾱ_t = ∏ α_s
        self.alphas_cumprod_prev = torch.cat([                   # ᾱ_{t-1}
            torch.tensor([1.0]), self.alphas_cumprod[:-1]
        ])
        
        # Forward process: q(x_t | x_0)
        self.sqrt_alphas_cumprod = self.alphas_cumprod.sqrt()
        self.sqrt_one_minus_alphas_cumprod = (1. - self.alphas_cumprod).sqrt()
        
        # Reverse process: varianza fija β̃_t
        # β̃_t = (1 - ᾱ_{t-1}) / (1 - ᾱ_t) · β_t
        self.posterior_variance = (
            self.betas * (1. - self.alphas_cumprod_prev) 
                       / (1. - self.alphas_cumprod)
        )
        
        # Para reconstruir x_0 desde x_t y ε
        self.sqrt_recip_alphas_cumprod = (1. / self.alphas_cumprod).sqrt()
        self.sqrt_recipm1_alphas_cumprod = (1. / self.alphas_cumprod - 1.).sqrt()


# ── Forward Process ────────────────────────────────────────────────────────

def q_sample(
    cfg: DiffusionConfig,
    x_0: torch.Tensor,          # (B, C, H, W) — imagen original
    t: torch.Tensor,             # (B,) — timesteps
    noise: torch.Tensor = None  # (B, C, H, W) — ruido a añadir
) -> Tuple[torch.Tensor, torch.Tensor]:
    """
    Muestrea q(x_t | x_0) usando la forma cerrada:
        x_t = sqrt(ᾱ_t) * x_0 + sqrt(1 - ᾱ_t) * ε
    
    Returns:
        x_t: imagen con ruido en el paso t
        noise: el ruido añadido (necesario para la loss)
    """
    if noise is None:
        noise = torch.randn_like(x_0)
    
    # Extraer coeficientes para cada elemento del batch
    # t puede ser diferente para cada imagen en el batch
    sqrt_alphas = cfg.sqrt_alphas_cumprod[t]               # (B,)
    sqrt_one_minus = cfg.sqrt_one_minus_alphas_cumprod[t]  # (B,)
    
    # Reshape para broadcasting: (B,) -> (B, 1, 1, 1)
    sqrt_alphas = sqrt_alphas[:, None, None, None]
    sqrt_one_minus = sqrt_one_minus[:, None, None, None]
    
    x_t = sqrt_alphas * x_0 + sqrt_one_minus * noise
    return x_t, noise


# ── Visualización del Forward Process ────────────────────────────────────────

def visualize_forward_process(x_0: torch.Tensor, cfg: DiffusionConfig,
                               timesteps: list = [0, 100, 300, 500, 700, 900, 999]):
    """Visualiza cómo el ruido destruye la imagen gradualmente."""
    fig, axes = plt.subplots(1, len(timesteps), figsize=(18, 3))
    
    # Usamos el mismo ruido base para mostrar la degradación continua
    noise = torch.randn_like(x_0)
    
    for ax, t in zip(axes, timesteps):
        if t == 0:
            img = x_0
        else:
            t_tensor = torch.tensor([t])
            img, _ = q_sample(cfg, x_0.unsqueeze(0), t_tensor, 
                               noise.unsqueeze(0))
            img = img.squeeze(0)
        
        # Para visualización: normalizar a [0, 1]
        img_np = img.permute(1, 2, 0).numpy()
        img_np = (img_np - img_np.min()) / (img_np.max() - img_np.min() + 1e-8)
        
        ax.imshow(img_np.squeeze(), cmap='gray' if img_np.shape[-1] == 1 else None)
        ax.set_title(f't = {t}\nᾱ_t = {cfg.alphas_cumprod[t]:.3f}', fontsize=9)
        ax.axis('off')
    
    plt.suptitle('Forward Process: Destrucción gradual de la imagen', 
                  fontsize=12, y=1.02)
    plt.tight_layout()
    plt.show()

2. Arquitectura de la red neuronal (U-Net simplificado)

import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class SinusoidalTimeEmbedding(nn.Module):
    """
    Embedding de tiempo sinusoidal (como en Transformers).
    Mapea el scalar t a un vector de dimensión dim.
    
    La red necesita saber en qué "nivel de ruido" está para 
    predecir el ruido correctamente.
    """
    def __init__(self, dim: int):
        super().__init__()
        self.dim = dim
        
    def forward(self, t: torch.Tensor) -> torch.Tensor:
        # t: (B,) enteros
        device = t.device
        half_dim = self.dim // 2
        
        # Frecuencias logarítmicamente espaciadas
        embeddings = math.log(10000) / (half_dim - 1)
        embeddings = torch.exp(torch.arange(half_dim, device=device) * -embeddings)
        
        # (B, half_dim)
        embeddings = t[:, None].float() * embeddings[None, :]
        
        # Concatenar sin y cos: (B, dim)
        embeddings = torch.cat([embeddings.sin(), embeddings.cos()], dim=-1)
        return embeddings


class ResidualBlock(nn.Module):
    """
    Bloque residual con condicionamiento en el tiempo.
    time_emb se añade como sesgo (FiLM conditioning simplificado).
    """
    def __init__(self, in_channels: int, out_channels: int, time_emb_dim: int):
        super().__init__()
        self.norm1 = nn.GroupNorm(8, in_channels)
        self.conv1 = nn.Conv2d(in_channels, out_channels, 3, padding=1)
        
        self.norm2 = nn.GroupNorm(8, out_channels)
        self.conv2 = nn.Conv2d(out_channels, out_channels, 3, padding=1)
        
        # Proyección del tiempo (actúa como sesgo espacial)
        self.time_proj = nn.Linear(time_emb_dim, out_channels)
        
        # Skip connection si cambia el número de canales
        self.residual_conv = (
            nn.Conv2d(in_channels, out_channels, 1)
            if in_channels != out_channels else nn.Identity()
        )
        self.act = nn.SiLU()  # Swish activation
    
    def forward(self, x: torch.Tensor, t_emb: torch.Tensor) -> torch.Tensor:
        h = self.act(self.norm1(x))
        h = self.conv1(h)
        
        # Añadir embedding de tiempo
        t_bias = self.time_proj(self.act(t_emb))  # (B, out_channels)
        h = h + t_bias[:, :, None, None]          # broadcast espacial
        
        h = self.act(self.norm2(h))
        h = self.conv2(h)
        
        return h + self.residual_conv(x)


class SimpleUNet(nn.Module):
    """
    U-Net simplificado para predecir el ruido ε_θ(x_t, t).
    
    Arquitectura:
        Encoder: [C] -> [2C] -> [4C]
        Bottleneck: [4C]
        Decoder: [4C] -> [2C] -> [C]
    
    Con skip connections en cada nivel.
    """
    def __init__(
        self, 
        in_channels: int = 1,
        base_channels: int = 64,
        time_emb_dim: int = 256,
    ):
        super().__init__()
        C = base_channels
        
        # Embedding de tiempo
        self.time_mlp = nn.Sequential(
            SinusoidalTimeEmbedding(C),
            nn.Linear(C, time_emb_dim),
            nn.SiLU(),
            nn.Linear(time_emb_dim, time_emb_dim),
        )
        
        # ── Encoder ──────────────────────────────────────────────
        self.conv_in = nn.Conv2d(in_channels, C, 3, padding=1)
        
        self.enc1 = ResidualBlock(C, C, time_emb_dim)
        self.down1 = nn.Conv2d(C, C, 4, stride=2, padding=1)      # /2
        
        self.enc2 = ResidualBlock(C, 2*C, time_emb_dim)
        self.down2 = nn.Conv2d(2*C, 2*C, 4, stride=2, padding=1)  # /4
        
        # ── Bottleneck ────────────────────────────────────────────
        self.mid1 = ResidualBlock(2*C, 4*C, time_emb_dim)
        self.mid2 = ResidualBlock(4*C, 4*C, time_emb_dim)
        
        # ── Decoder (con skip connections) ─────────────────────────
        self.up2 = nn.ConvTranspose2d(4*C, 2*C, 4, stride=2, padding=1)
        self.dec2 = ResidualBlock(4*C, 2*C, time_emb_dim)  # 4C porque recibe skip
        
        self.up1 = nn.ConvTranspose2d(2*C, C, 4, stride=2, padding=1)
        self.dec1 = ResidualBlock(2*C, C, time_emb_dim)    # 2C porque recibe skip
        
        # ── Salida ────────────────────────────────────────────────
        self.norm_out = nn.GroupNorm(8, C)
        self.conv_out = nn.Conv2d(C, in_channels, 3, padding=1)
        
        # Inicializar la última capa a cero (mejor estabilidad inicial)
        nn.init.zeros_(self.conv_out.weight)
        nn.init.zeros_(self.conv_out.bias)
    
    def forward(self, x_t: torch.Tensor, t: torch.Tensor) -> torch.Tensor:
        """
        Args:
            x_t: (B, C, H, W) — imagen ruidosa en paso t
            t:   (B,)         — timesteps como enteros
        Returns:
            epsilon_pred: (B, C, H, W) — ruido predicho
        """
        # Time embedding
        t_emb = self.time_mlp(t)  # (B, time_emb_dim)
        
        # Encoder
        x = self.conv_in(x_t)
        h1 = self.enc1(x, t_emb)          # skip1
        h1_down = self.down1(h1)
        
        h2 = self.enc2(h1_down, t_emb)    # skip2
        h2_down = self.down2(h2)
        
        # Bottleneck
        h = self.mid1(h2_down, t_emb)
        h = self.mid2(h, t_emb)
        
        # Decoder con skip connections
        h = self.up2(h)
        h = torch.cat([h, h2], dim=1)     # concatenar skip2
        h = self.dec2(h, t_emb)
        
        h = self.up1(h)
        h = torch.cat([h, h1], dim=1)     # concatenar skip1
        h = self.dec1(h, t_emb)
        
        # Salida
        h = F.silu(self.norm_out(h))
        return self.conv_out(h)

import torch
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
from tqdm import tqdm


# ── Loss Function ─────────────────────────────────────────────────────────

def ddpm_loss(
    model: SimpleUNet,
    cfg: DiffusionConfig,
    x_0: torch.Tensor,           # (B, C, H, W)
    device: str = 'cpu'
) -> torch.Tensor:
    """
    Computa la pérdida simplificada de DDPM:
    
        L_simple = E_{t, x_0, ε} [ ||ε - ε_θ(x_t, t)||² ]
    
    Pasos:
        1. Muestrear t uniformemente de {1, ..., T}
        2. Muestrear ruido ε ~ N(0, I)
        3. Calcular x_t = sqrt(ᾱ_t) x_0 + sqrt(1-ᾱ_t) ε
        4. Predecir ε con la red
        5. MSE entre ε predicho y real
    """
    B = x_0.shape[0]
    
    # Paso 1: timestep aleatorio para cada elemento del batch
    t = torch.randint(0, cfg.T, (B,), device=device, dtype=torch.long)
    
    # Paso 2: ruido gaussiano
    epsilon = torch.randn_like(x_0)
    
    # Paso 3: forward process — crear x_t desde x_0 directamente
    x_t, _ = q_sample(cfg, x_0, t, noise=epsilon)
    
    # Paso 4: predecir el ruido
    epsilon_pred = model(x_t, t)
    
    # Paso 5: MSE
    loss = F.mse_loss(epsilon_pred, epsilon)
    return loss


# ── Reverse Process / Sampling ─────────────────────────────────────────────

@torch.no_grad()
def p_sample(
    model: SimpleUNet,
    cfg: DiffusionConfig,
    x_t: torch.Tensor,    # (B, C, H, W)
    t: int,               # timestep actual (escalar)
    device: str = 'cpu'
) -> torch.Tensor:
    """
    Un paso del reverse process: muestrea p_θ(x_{t-1} | x_t).
    
    Algoritmo:
        1. Predecir ε_θ(x_t, t)
        2. Calcular μ_θ(x_t, t) = 1/√α_t · (x_t - β_t/√(1-ᾱ_t) · ε_θ)
        3. Muestrear x_{t-1} = μ_θ + σ_t · z,  z ~ N(0,I)  (si t > 0)
    """
    B = x_t.shape[0]
    t_tensor = torch.full((B,), t, device=device, dtype=torch.long)
    
    # Predecir ruido
    epsilon_pred = model(x_t, t_tensor)
    
    # Extraer coeficientes
    alpha_t = cfg.alphas[t]
    alpha_cumprod_t = cfg.alphas_cumprod[t]
    beta_t = cfg.betas[t]
    
    # Media del reverse step
    coef = beta_t / (1 - alpha_cumprod_t).sqrt()
    mu_theta = (1 / alpha_t.sqrt()) * (x_t - coef * epsilon_pred)
    
    if t == 0:
        return mu_theta  # último paso: sin ruido
    
    # Varianza β̃_t
    posterior_var = cfg.posterior_variance[t]
    z = torch.randn_like(x_t)
    
    return mu_theta + posterior_var.sqrt() * z


@torch.no_grad()
def p_sample_loop(
    model: SimpleUNet,
    cfg: DiffusionConfig,
    shape: tuple,          # (B, C, H, W)
    device: str = 'cpu'
) -> torch.Tensor:
    """
    Loop completo del reverse process: genera imágenes desde ruido puro.
    Itera T pasos desde x_T ~ N(0, I) hasta x_0.
    """
    model.eval()
    
    # Paso 1: muestrear x_T ~ N(0, I) — punto de partida
    x = torch.randn(shape, device=device)
    
    # Pasos 2-T+1: iterar el reverse process
    for t in tqdm(reversed(range(cfg.T)), desc='Sampling', total=cfg.T):
        x = p_sample(model, cfg, x, t, device)
    
    return x  # x_0 — imágenes generadas


# ── Training Loop ─────────────────────────────────────────────────────────

def train_ddpm(
    epochs: int = 10,
    batch_size: int = 64,
    lr: float = 2e-4,
    device: str = 'cuda' if torch.cuda.is_available() else 'cpu'
):
    print(f"Training on {device}")
    
    # Dataset: MNIST como ejemplo
    transform = transforms.Compose([
        transforms.ToTensor(),
        transforms.Normalize([0.5], [0.5]),  # -> [-1, 1]
    ])
    dataset = datasets.MNIST('.', download=True, transform=transform)
    loader = DataLoader(dataset, batch_size=batch_size, shuffle=True, 
                        num_workers=4, pin_memory=True)
    
    # Modelo y configuración
    cfg = DiffusionConfig(T=1000, schedule='linear')
    model = SimpleUNet(in_channels=1, base_channels=64).to(device)
    optimizer = optim.AdamW(model.parameters(), lr=lr)
    scheduler = optim.lr_scheduler.OneCycleLR(
        optimizer, max_lr=lr, 
        steps_per_epoch=len(loader), epochs=epochs
    )
    
    # Mover coeficientes al device
    for attr in ['alphas', 'alphas_cumprod', 'betas', 'posterior_variance',
                 'sqrt_alphas_cumprod', 'sqrt_one_minus_alphas_cumprod']:
        setattr(cfg, attr, getattr(cfg, attr).to(device))
    
    model.train()
    for epoch in range(epochs):
        total_loss = 0.0
        
        for x_0, _ in tqdm(loader, desc=f'Epoch {epoch+1}/{epochs}'):
            x_0 = x_0.to(device)
            optimizer.zero_grad()
            
            # Computar L_simple
            loss = ddpm_loss(model, cfg, x_0, device)
            
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()
            scheduler.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(loader)
        print(f"Epoch {epoch+1}: Loss = {avg_loss:.4f}")
        
        # Generar algunas muestras cada 5 epochs
        if (epoch + 1) % 5 == 0:
            samples = p_sample_loop(model, cfg, (16, 1, 28, 28), device)
            # samples están en [-1, 1], convertir a [0, 1] para visualizar
            samples = (samples.clamp(-1, 1) + 1) / 2
            # ... guardar o visualizar samples
    
    return model, cfg

4. Verificación: el ELBO en práctica

elbo_verification.py Python · PyTorch — verificación matemática 
import torch
import torch.distributions as dist


def compute_kl_gaussians(mu1, sigma1_sq, mu2, sigma2_sq):
    """
    KL(N(μ₁, σ₁²I) || N(μ₂, σ₂²I)) en forma cerrada.
    Para gaussianas diagonales:
        KL = 0.5 * [log(σ₂²/σ₁²) - 1 + σ₁²/σ₂² + (μ₁-μ₂)²/σ₂²]
    """
    return 0.5 * (
        (sigma2_sq / sigma1_sq).log() 
        - 1 
        + sigma1_sq / sigma2_sq 
        + (mu1 - mu2)**2 / sigma2_sq
    )


def compute_L_T(cfg, x_0):
    """
    L_T = KL(q(x_T|x_0) || N(0,I))
    
    q(x_T|x_0) = N(sqrt(ᾱ_T) x_0, (1-ᾱ_T) I)
    
    Esperamos que esto sea ≈ 0 para T bien elegido.
    """
    alpha_bar_T = cfg.alphas_cumprod[-1]
    
    mu_q = (alpha_bar_T.sqrt() * x_0).mean()
    var_q = 1 - alpha_bar_T
    
    # KL con N(0,1)
    kl = compute_kl_gaussians(
        mu1=mu_q, sigma1_sq=var_q,
        mu2=torch.tensor(0.0), sigma2_sq=torch.tensor(1.0)
    )
    return kl


def compute_L_t_analytical(cfg, t: int, epsilon_error_norm_sq: float):
    """
    L_{t-1} = KL(q(x_{t-1}|x_t,x_0) || p_θ(x_{t-1}|x_t))
    
    Dado que ambas son gaussianas con varianza β̃_t, esto se reduce a:
        L_{t-1} ∝ (β_t² / (2 α_t (1-ᾱ_t) β̃_t)) · ||ε - ε_θ||²
    
    El término simplificado (sin los pesos) es simplemente el MSE.
    
    Params:
        epsilon_error_norm_sq: ||ε - ε_θ||² promedio
    """
    beta_t = cfg.betas[t]
    alpha_t = cfg.alphas[t]
    alpha_bar_t = cfg.alphas_cumprod[t]
    beta_tilde = cfg.posterior_variance[t]
    
    # Peso teórico del término L_{t-1}
    weight = beta_t**2 / (2 * alpha_t * (1 - alpha_bar_t) * beta_tilde)
    
    return {
        'weight': weight.item(),
        'L_t_weighted': (weight * epsilon_error_norm_sq).item(),
        'L_t_simple': epsilon_error_norm_sq,  # lo que optimizamos
    }

    Demuestra numéricamente la magnitud de cada componente del ELBO.
    """
    cfg = DiffusionConfig(T=1000, schedule='linear')
